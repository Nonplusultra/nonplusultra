How to... wie erstelle ich eine Website?

Ganz am Anfang stellt sich natürlich die Frage, sind bereits Kenntnisse vorhanden? Wenn das nicht der Fall sein sollte, verweise ich gern auf meine Rubrik Codecademy. Desweiteren gibt es verschiedene Lösungswege um seine eigene Seite ins Netz hochzuziehen. Da ich diese Seite in meinem Studium erstellen musste, berichte ich über meinen (Leidens-)weg. ;) Mit der VirtualBox und einer Ubuntu-Distribution fing alles an: damit Linux-Einsteiger nicht komplett überfordert sind, lernten wir zunächst via Codecademy die "Command Line". Damit war es uns möglich einige wichtige Konsolenbefehle auszuführen. Mit diesen Grundkenntnissen konnten wir uns dann git nähern. Git ist ein Versionsverwaltungsprogramm. Kurz und bündig: Wir haben unsere Dateien im Blick, wissen wann, was und hoffentlich auch wie etwas verändert wurde. Zu Git könnte man jetzt eine eigene Seite schreiben. (Vielleicht kommt das auch noch?) Mit Git können wir jedenfalls unsere HTML-Dateien, die CSS und was man nicht noch alles auf seiner Homepage haben möchte, bestens im Blick behalten. Wir starten: Zunächst mal brauchen wir Seiten, damit die Homepage nicht ganz so leer wird, dafür habe ich Eclipse verwendet. Dann benötigt man noch Bootstrapin irgendeiner Variante (hier via CDN). Bootstrap ist eine Art Baukasten für eine Homepage. Wichtige Funktionalitäten, Standards, die man verwendet, sind schon vorgefertigt werden und können gleich implmentiert werden. Jetzt muss man nur noch seine Seiten füllen und alles so anpassen, damit man zufrieden ist. Hilfe findet man genügend im Web. Ob man noch mal bei Codecademy vorbeischaut oder einfach wild Begriffe in die Google-Suchmaschine eingibt, irgendwann kommt man auf eine Antwort (selbst wenn nur indirekt).

  

Zum Schluss müssen wir noch einen Bugfix oder ein zusätzliches Feature via "Pull Request" auf GitHub übergeben. Ich bekomme ebenfalls einen und muss diesesn übernehmen.
